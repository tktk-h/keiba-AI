# 中央競馬 期待値予想プログラム 設計書 (v1)

## 目的
JRA(中央競馬)のレースについて、出走馬の過去成績・オッズ・血統・調教情報などをもとに各馬の確率を推定し、単勝・複勝・馬連・三連複など複数券種を比較して期待値が最も高くなる買い方(組み合わせ)を提案する。

## 開発方針
段階的開発。v1ではシンプルな統計モデル(ロジスティック回帰)から始め、データが蓄積された段階で機械学習モデル(LightGBM等)への差し替えを検討する。

## 技術スタック
Python

## 全体構成

1. **データ収集モジュール**
   netkeibaから対象レースIDを指定して、出走馬情報・過去成績・血統・調教情報を取得し、CSV/JSONで保存する。

2. **特徴量作成モジュール**
   収集データから予測モデル用の特徴量を作成する。

3. **予測モジュール**
   ロジスティック回帰により各馬の1着確率を推定し、プレイス確率モデル(ハーヴィル法等)で2着・3着確率を近似する。

4. **期待値計算モジュール**
   推定確率とオッズから各券種・組み合わせの期待値を計算する。

5. **推奨出力モジュール**
   期待値が高い買い方をランキング表示する。

## 収集データ(v1)

### レース情報
- レース名、開催日
- コース(競馬場・距離・芝/ダート・回り)
- 馬場状態、天候

### 出走馬情報(当該レース)
- 馬名、性別・年齢、斤量
- 騎手、枠番・馬番
- 単勝オッズ、人気順位
- 馬体重・増減
- 脚質

### 各馬の過去成績(直近5走)
- 着順、レース日
- 競馬場・距離・芝ダート・馬場状態
- 走破時計、上がり3F
- 人気、斤量、騎手
- レースクラス(条件)

### 血統情報
- 父・母・母父

### 調教情報
- 直前の調教タイム・コース・追切評価

v1では上記データの取得・保存までを実装し、モデルへの組み込みは段階的に拡張する。

## 特徴量(v1)
- 過去5走の着順、走破時計(or 上がり3F)、人気、距離・コース適性
- 当該レースの単勝オッズ
- 馬の基本情報(性別・年齢・斤量)

## 予測モデル(v1)
- ロジスティック回帰で「1着になる確率」を推定
- プレイス確率モデル(ハーヴィル法等)で2着・3着確率を近似算出
- 学習データは過去レース結果をスクレイピングして蓄積し、随時追加

## 期待値計算
- 単勝・複勝: 推定確率 × オッズ - 1
- 馬連・三連複等: 各馬の確率から組み合わせ確率を推定(独立性を仮定した簡易計算からスタート)し、対応オッズと掛けて期待値算出
- 全券種・全組み合わせの期待値を計算し、期待値の高い順にランキング表示(例: 期待値1.0以上、上位5件)

## v1のスコープ
- 対象は特定のレースIDを1つ指定して処理する形とする
- 将来的に開催日指定で全レースを処理する形に拡張する

## エラー処理・テスト
- スクレイピング失敗時(ページ構造変更・通信エラー)はエラーメッセージを出して中断する
- 特徴量作成・期待値計算モジュールは単体テストでサンプルデータを使って検証する
- v1ではモデル精度のバックテスト評価は対象外とし、将来バージョンで対応する
